2. Implement the "Coder Agent"This agent's job is to write a Python function based on a user's request.Write the generate_code function: This function will take a text specification (e.g., "write a function to sum a list of numbers") and return a string containing the Python code.API-Powered Generation: The primary method should be calling an LLM like Gemini or GPT-4o. Use a clear, role-based prompt.Example Prompt:
"You are an expert Python developer. Your task is to write a single Python function that meets the following specification. Do not include any text, explanations, or example usage other than the code for the function itself.

Specification:
{user's request here}"
3. Implement the "Tester Agent"This is a new, crucial agent. Its role is to write a comprehensive set of unit tests for the code generated by the Coder Agent.Write the generate_tests function: This function will take the original user specification and the generated code as input, and it should return a string containing Python code for unittest test cases.Prompt the LLM to Think About Edge Cases: The key to a good Tester Agent is a prompt that encourages it to be thorough.Example Prompt:
"You are a meticulous Quality Assurance engineer. Your task is to write a Python `unittest` test suite for the following function. Be comprehensive and include tests for edge cases, such as empty inputs, invalid inputs, and different data types.

Original Request: '{user's request here}'

Function to Test:
{generated code here}

Return only the Python code for the `unittest.TestCase` class."
4. Implement the "Reviewer Agent"This agent acts as the final judge, executing the tests and checking the code quality.Write the review_code function: This function will now take both the generated code and the generated tests as input.Execute Tests Safely: This is the most critical step. Combine the generated function and the generated unittest code into a single script.Safety First: Dynamically executing AI-generated code is risky. Use exec() within a controlled, sandboxed environment to run the combined script. Capture the output from the unittest framework to see which tests passed or failed.Check for Quality (Linters): Use libraries like pylint or flake8 to programmatically check the original code for style errors, bugs, and code smells.Construct Feedback: Combine the unit test results and the linter feedback into a single, clear report.5. Wire Up the Full Multi-Agent PipelineNow, create a main function that orchestrates the collaboration between your agents.Get the task description from the user.Call the Coder Agent (generate_code) to produce the Python function.Call the Tester Agent (generate_tests), feeding it the original request and the generated code to produce the unit tests.Call the Reviewer Agent (review_code) with both the code and the tests.Display the function, the tests that were written for it, and the final review report to the user.(Optional Power-Up) Create a "fix it" button. If the review shows errors, send the code, tests, and feedback report back to the Coder Agent and ask it to provide a corrected version.6. Build an Interactive User InterfaceUse Gradio to create a simple and effective UI for your project.Essential Components:A Textbox for the user's programming task.A "Generate & Review" Button.Three output components arranged side-by-side or in tabs:A Textbox (set to language="python") for the Generated Code.A Textbox (set to language="python") for the Generated Tests.A Textbox or Markdown component for the Final Review.
