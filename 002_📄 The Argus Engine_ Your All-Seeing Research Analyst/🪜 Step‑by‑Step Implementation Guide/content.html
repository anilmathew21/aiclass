2. Ingest the "Case File" and Create the Knowledge BaseYour first step is to process the user's collection of documents and build a unified, searchable knowledge base.Handle Multiple Documents: The system must accept a collection of files (e.g., multiple PDFs). A simple way to manage this in Colab is to have the user upload all their files to a temporary directory.Load and Parse with Metadata: Use a loader like SimpleDirectoryReader from LlamaIndex. When it loads documents, it automatically creates metadata for each one (like the file name), which is crucial for citations later. The framework will handle the parsing and chunking for you.Create the Vector Index: Once the documents are loaded, create a single VectorStoreIndex from them. This process automatically converts all text chunks into vector embeddings and stores them in a searchable index. This unified index is your AI's "brain."3. Implement the Cross-Examination Logic (RAG)This is where your system becomes an investigator. You will configure the RAG pipeline to synthesize answers from all available sources.Configure the Query Engine: Create a query engine from your index. LlamaIndex's default query engine is already designed to retrieve relevant context from across all the documents in the index.Engineer a Synthesis Prompt: The magic is in the prompt. You need to instruct the LLM to act as a research analyst, comparing and contrasting information and, most importantly, citing its sources. You can customize the prompt used by the query engine.Example Prompt:
"You are an expert research analyst. Based *only* on the provided context from multiple sources, answer the user's question. Your answer must synthesize information from the different sources. When you use information from a source, you MUST cite it using its filename (e.g., ).

Context:
[Context from multiple documents will be inserted here by the RAG pipeline]

Question:
[User's question here]"
Query and Get a Cited Response: When the user asks a question, the query engine will find the most relevant chunks from all documents, insert them into your prompt, and send it to the LLM to get a synthesized, cited answer.4. Build the Investigator's DashboardCreate a user interface with Gradio or Streamlit that is intuitive for a research task.Multi-File Upload: Use a file component that allows users to upload their entire "case file" at once.Clear Q&A: Include a text input for the user to ask their complex, comparative questions.Display Synthesized, Cited Answers: The output should be a clean Markdown or Textbox component that presents the synthesized answer and clearly shows the citations, so the user can trust and verify the information.5. Iterate and DocumentTurn a good project into a great one with refinement and clear documentation.Experiment with Prompts: Tweak your synthesis prompt to get better, more detailed answers. Can you ask it to identify conflicts between sources?Refine Retrieval: Experiment with changing the number of retrieved chunks to see how it affects the quality of the synthesis.Document Your Architecture: In your Colab notebook, use text cells to explain your RAG pipeline. Detail how you managed citations and showcase the prompts you engineered. This documentation is a critical part of your submission.
