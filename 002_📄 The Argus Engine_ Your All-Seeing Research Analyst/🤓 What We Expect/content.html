For this hackathon, your goal is to build a functional prototype of The Argus Engine. We are looking for a system that can successfully ingest multiple documents and perform sophisticated, cross-document analysis to answer complex questions.A successful project will:1. Ingest a "Case File" of Multiple Documents:Your UI must allow a user to upload a collection of documents (at least 2-3) at the same time. This collection represents the "case file" for the investigation.2. Create a Unified, Citable Knowledge Base:Your system must process all uploaded documents, chunk them, and create vector embeddings.Crucially, you must store these embeddings in a single vector index while also keeping metadata that tracks the original source of each chunk (e.g., "Page 5 of Document A").3. Perform True Cross-Document Synthesis:Your application must be able to answer questions that require synthesizing information from more than one of the uploaded documents. The user should be able to ask comparative or thematic questions, such as:"Compare the main arguments from 'Article 1' and 'Report B'.""Create a timeline of events using information from all sources.""What are the recurring themes mentioned across all of these documents?"4. Provide Answers with Verifiable Citations:This is a key requirement. The answers generated by your LLM must cite which document (and ideally, which part of the document) the information was pulled from. For example: “According to Report B, the project started in March, while Article 1 suggests a later start date in April.”5. Present Findings in a Clear "Investigator's Dashboard":Build an interface using Gradio or Streamlit where a user can upload their case file, ask their complex question, and view the synthesized, cited answer.6. Document Your RAG Architecture:Deliver a well-commented Colab notebook that explains your design. Detail how you managed metadata for citations, the structure of your RAG pipeline, and the prompts you engineered to encourage synthesis.The goal is to move beyond a simple Q&A bot. A winning prototype will demonstrate the power of RAG to not just find facts, but to compare, contrast, and create new insights from a collection of documents.
